# V0 基础：先让视觉系统可观察、可复现

## 1. 视觉程序不等于“打开相机识别目标”

完整视觉系统至少要回答：这张图是谁拍的、什么时候拍的、算法用了多久、有没有检测到、坐标
属于哪个坐标系、结果是否过期，以及失败时 MCU 应该做什么。

因此 V0 先建立 Frame、Result、回放和协议。它暂时不识别圆形，但能保证以后圆形算法出现问题
时，我们知道输入、配置、结果和错误分别是什么。

## 2. Frame 是带身份证的图片

图片数组只有像素。`Frame` 还包含：

- `frame_id`：第几帧。
- `captured_monotonic_ns`：采集时间。
- `source`：来自文件、MaixCAM 还是树莓派。
- `pixel_format`：GRAY/RGB/BGR。
- `metadata`：原文件、曝光等附加信息。

Frame 发布后设为只读，避免录像线程或叠加框代码偷偷修改算法正在处理的原图。

## 3. 为什么不用 `0,0` 表示没检测到

图像中心可能真的就是 `(0,0)`，所以坐标不能同时表示“居中”和“故障”。正式结果使用：

```text
valid=false                 没有可用目标
valid=true + x/y            有合法目标
fault_flags                 为什么不可用或系统哪里异常
coordinate_frame            x/y 属于哪种坐标
confidence                  算法有多确信
result_age_ms               结果有多旧
```

比赛时 MCU 先看 valid、年龄和置信度，再决定是否使用坐标。

## 4. 为什么先做离线回放

实时相机画面每次都不同，光线、位置和运动也在变化。直接调实时画面时，很难证明是代码变好，
还是这次目标刚好更清楚。

离线回放把同一批图片反复送入算法，使每次只改变一个参数。这样可以比较误检、漏检、耗时，
也能把赛场失败帧带回电脑复现。

## 5. latest-frame 是什么

如果相机 60 FPS，算法只能处理 30 FPS，无界队列会越积越多。算法最终处理的是几秒前的图，
即使识别正确也会让云台追错位置。

latest-frame 槽容量只有 1：新帧到来时覆盖尚未处理的旧帧，并记录 overwritten 数量。它牺牲
部分帧，换取结果始终尽量新鲜。对于控制类视觉，通常“新鲜的少量帧”比“完整但过期的帧”
更有价值。

## 6. CRC 能解决什么，不能解决什么

CRC 可以发现串口传输中的坏字节和截断，但不能判断目标是否真的识别正确，也不能代替结果
年龄、sequence 和通信超时。

协议因此同时使用：

- SOF/length：找到帧边界。
- sequence：发现丢帧或重复。
- timestamp/age：判断结果是否过期。
- valid/confidence/fault：判断结果能不能用。
- CRC16：判断字节是否损坏。

## 7. 测试通过分别意味着什么

- 单元测试通过：某个函数对构造输入符合预期。
- 离线回放通过：同一批文件能稳定处理。
- MaixCAM 部署通过：设备上能启动和读取相机。
- UART 通过：视觉与 MCU 的真实线缆和协议可用。
- 整机通过：安全门、控制和真实动作共同通过。

V0 只完成前两层中的基础数据流，没有证明任何目标识别能力。

## 8. 以后遇到问题先看哪里

```text
没有图像 -> CameraSource / 接线 / 曝光
有图像无目标 -> 算法 / ROI / 阈值 / 模型
结果正确但动作错误 -> 坐标系 / 标定 / MCU 控制
偶发卡顿 -> 队列 / 日志 / 网络 / 录像 / 温度
串口数据乱 -> 帧边界 / 长度 / sequence / CRC / 电平与共地
```

先确定问题属于哪一层，再改参数或代码，不要同时调整相机、算法和控制器。

## 9. 为什么“同一条测试命令”可能得到不同结果

电脑里可能同时有多个 Python。PowerShell 输入 `python` 时，会选择 PATH 中排在最前面的版本，
不一定是工程需要的版本。先检查：

```powershell
python --version
(Get-Command python).Source
```

本工程采用 src-layout，源码在 `src/echo_vision`。如果还没有执行 `pip install -e .`，测试前需把
`src` 加到模块搜索路径，或使用已经安装项目的虚拟环境：

```powershell
$env:PYTHONPATH = (Resolve-Path 'src').Path
python -m unittest discover -s tests -v
```

看到语法错误时先确认 Python 版本，看到 `ModuleNotFoundError: echo_vision` 时先确认安装状态或
`PYTHONPATH`。这些属于工具链问题，不代表视觉算法本身一定有错。
