# 学习日志规则与索引

本目录面向没有视觉基础的队员，解释工程中遇到的问题、原理和排障方法。学习日志不是简单
复制源码或命令，而要让读者理解“为什么这样设计”和“比赛时怎么判断问题在哪一层”。

## 新任务阅读规则

新任务先读本索引，再读与当前模块相关的学习日志。第一次接手视觉工程至少阅读：

1. `VISION_LEARNING_PATH.md`
2. `2026-07-15_V0_FOUNDATIONS.md`
3. `2026-07-15_ROBUSTNESS_FOUNDATIONS.md`

以后每增加一种重要能力，应在此处增加索引，例如曝光、轮廓、单应性、跟踪、模型和 UART。

## 写作要求

- 先用日常语言解释，再给专业术语。
- 给出错误示例、正确思路和怎样观察结果。
- 说明它与真实赛题、硬件和安全的关系。
- 引用正式源码和文档位置，但不要求读者先看完源码。
- 区分“理解原理”“本机验证”“硬件验证”和“整车验证”。
- 每个重要 debuglog 至少提炼一条可学习结论；没有新知识时说明原因。
