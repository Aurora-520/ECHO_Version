系统版本、资源、相机、IMU、UART、服务和旧模型状态，并固化为可交接证据。

- V0 与鲁棒性文档已提交并同步 GitHub，工作区干净。
- 用户已将 MaixCAM Pro 通过 USB-C 接入电脑。

- 只读枚举 Windows NCM/RNDIS、路由、ARP、串口和 TCP 端口。

- 读取 `/boot/ver`、`/mnt/data/sensor_cfg.ini`、MaixPy dist-info、`/maixapp`、`/root/models`、
  `/proc`、`/sys` 和最近应用日志。
- 新建 V1 Phase、硬件/来源/旧代码/风险/学习/调试/能力和证据文档。
- 改进 `tools/discover_devices.ps1`，增加错误输出、设备 IP 推导和可选 SSH 探测。

## 问题与判断

- ExecutionPolicy/CIM 权限会让旧脚本失败，不能把空输出当作无设备。
- `import maix` 有 UART0 副作用，平台探测和纯算法环境必须避免。
- 设备时间明显错误，正式录像/数据集前必须校时。
- 当前 Settings 应用拥有相机和默认通信资源，暂不并行启动采集程序。
- 发现阶段并行 grep 暂时抬高负载，不能把该 top 快照当作空闲性能基线。

## 风险与下一步

- 先解决时间与 SSH 凭据管理，再部署可回滚的调试应用。
- 首次采集不加载模型、不发 UART，只验证 raw、元数据、健康和资源。
