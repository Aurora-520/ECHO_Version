# 2026-07-15 V0 基线调试复盘

## 背景与影响

V0 需要在空工程中建立可运行、可测试和可交接的视觉底座。虽然没有真实视觉算法和硬件，
工程初始化、环境依赖、配置、时间测量、错误报告和序列化问题会直接影响以后所有模块。

## 问题一：空 `.git` 目录不是有效仓库

### 现象

目录存在 `.git`，但 `git status` 报告不是 Git 仓库；普通 `git init` 又因 `.git` 写权限失败。

### 根因

原 `.git` 只是空目录，缺少 HEAD、config、objects 等元数据；当前执行环境对 `.git` 有单独
写权限门禁。

### 修复与验证

确认用户授权后，使用 bundled Git 在明确路径 `E:\ECHO_Version` 初始化 `main`。随后
`git status`、提交和 annotated tag 均正常。

### 如何避免复发

- 开工时必须执行 `git status`，不能仅凭 `.git` 文件夹存在判断仓库有效。
- Git 元数据写入失败要区分权限问题和仓库内容问题。
- 禁止因权限问题另建第二个“正式工程”。

## 问题二：计划环境与实际 Python 依赖不同

### 现象

计划使用 Python 3.11/OpenCV/pytest，但本机 Codex runtime 实际为 Python 3.12.13，只有
NumPy/Pillow，没有 OpenCV、pytest、PyYAML 和 pyserial。

### 根因

设计阶段的目标工具链不等于当前执行环境。若代码在模块导入时强制加载 OpenCV/Maix/串口，
无硬件回放也会失败。

### 修复与验证

- V0 最小文件源使用 NumPy/Pillow。
- OpenCV、Maix 和 pyserial 采用延迟导入。
- 测试使用标准库 `unittest`，14/14 通过。

### 如何避免复发

每个平台开发前先记录 Python、库和设备 API 版本。纯算法不得因为设备依赖未安装而无法导入。

### 对真实视觉任务的影响

以后相同算法可以先在电脑回放，再部署到 MaixCAM/树莓派；设备环境故障不会阻断算法回归。

## 问题三：slots 数据类默认值读取错误

### 现象

配置覆盖测试抛出 `source.patterns must be a list of strings`，即使没有提供 patterns。

### 根因

带 `slots=True` 的数据类字段通过 `SourceConfig.patterns` 访问时得到成员描述符，不是默认 tuple。

### 修复与验证

把默认 pattern 提取为模块常量 `DEFAULT_PATTERNS`，配置测试重新通过。

### 如何避免复发

配置默认值使用显式常量或实例，不依赖 slots 数据类的类字段描述符。每个默认配置都要有“仅
覆盖一个子字段”的测试。

## 问题四：极短回放得到 0 ms 和 0 FPS

### 现象

两张很小的测试图片回放完成，但单调时钟分辨率内开始/结束值相同，摘要 FPS 为 0。

### 根因

测量窗口短于当前平台时钟分辨率。0 不是算法真的没有速度，而是测量方法超出分辨率。

### 修复与验证

计算时将 elapsed ns 下限设为 1，避免除零；同时文档明确 no-op 的瞬时 FPS 不能作为性能指标。

### 如何避免复发

正式性能测试使用足够长的窗口、预热和 p50/p95，不用一两帧的瞬时结果评价算法。

## 问题五：输出路径失败但回放器吞掉异常

### 现象

CLI 向 `C:\tmp` 写 JSONL 时因当前沙箱权限失败。回放摘要只有 `failures=2`，没有失败原因。

### 根因

运行器为了继续批处理捕获了异常，但没有保留异常类型和消息，破坏了可诊断性。

### 修复与验证

- `ReplaySummary` 增加最多 10 条 `failure_messages`。
- 输出改到 workspace 内 ignored `artifacts`，两帧回放为 0 failure。

### 如何避免复发

任何 drop/failure 计数必须有来源分类和最近错误摘要。不能只有一个数字而没有上下文。

### 对真实视觉任务的影响

以后录像磁盘满、失败帧保存失败或网络异常时，主路径可以继续运行，同时健康状态能指出具体
旁路故障，不会误判为识别算法错误。

## 问题六：只读快照不能依赖通用深拷贝序列化

### 现象

核心类型使用只读 `MappingProxyType` 保存 attributes。通用 `dataclasses.asdict` 可能深拷贝
这些映射并失败。

### 根因

不可变运行时结构和通用递归深拷贝的语义不同。

### 修复与验证

为 Detection、Track、VisionResult 和 HealthSnapshot 编写显式序列化，新增带 attributes 的
测试并通过。

### 如何避免复发

协议和持久化格式必须使用显式 schema，不直接序列化内部对象。内部字段调整不能静默改变
对外协议。

## 可复用结论

- 环境能力必须在模块导入前发现，设备依赖应延迟加载。
- 所有性能数字必须匹配时钟分辨率和测量窗口。
- failure/drop 必须可分类、可定位。
- 内部不可变快照与外部 schema 分离。
- 这些规则适用于以后圆/矩形检测、录像、模型推理、UART 和调试服务。

## 仍未解决

- OpenCV、MaixCAM、树莓派相机、UART 和整机均未实测。
- V1 需要在真实设备上建立新的调试日志，不能沿用本日志宣称硬件通过。
