# ECHO_Vision 工程红线

1. 算法函数不得打开相机、读写 UART、访问 SSH/HTTP、录像或修改全局 Mission。
2. 同一物理相机只能有一个所有者，同一 UART 只能有一个所有者。
3. 视觉不得直接控制电机、底盘或云台；最终输出由 MSPM0G3507 唯一控制。
4. 高频采集/推理不得等待日志、网络、录像、UI、磁盘刷新或无界队列。
5. 队列必须有容量、drop/overflow/high-water 诊断和明确降级策略。
6. 所有外设等待必须有超时；相机故障、结果过期、低置信度和通信失联必须可见。
7. 禁止用 `0,0`、最大值、NaN 或其他合法数据值表达故障。
8. 模式切换必须停止旧 Pipeline、清理跟踪状态、发布无效结果，再启动新模式。
9. 参数更新先校验为 pending，在帧或 Pipeline 边界原子应用，不修改运行中的半份配置。
10. 模型必须登记来源、格式、输入、标签、SHA-256 和兼容平台；缺失时不得伪造。
11. 未保存原始日志或快照的数据不得事后补写。
12. 未部署不能写“MaixCAM 通过”，未接线不能写“UART 通过”，未动作不能写“整机通过”。
13. 普通应用部署不得升级为系统烧录；未知 USB 磁盘禁止写入。
14. 不修改只读参考工程，不复制整个开源仓库或旧单文件脚本进入正式工程。
15. 新聊天必须先读取状态和交接文档，不得从聊天印象继续硬件操作。
